GeoJSON Standardization & GIS Initialization

This document provides a comprehensive technical guide for processing GeoJSON data to optimize map rendering (Vector Tiles) and build routing networks (Routing Graph) integrated with the OpenStreetMap (OSM) infrastructure.

Part A

MBTiles Initialization (Vector Tiles)

Method 1: Automated Python Script Deployment

Execution Syntax (Terminal):

python geojson_to_mbtiles.py input.geojson roads.mbtiles
geojson_to_mbtiles.py
Click to Expand Script Click to Collapse Script 
import json
import subprocess
import os
import sys
import argparse

def main():
    parser = argparse.ArgumentParser(description="Convert GeoJSON to MBTiles")
    parser.add_argument("source", help="Input GeoJSON file path")
    parser.add_argument("output", help="Output MBTiles file path")
    args = parser.parse_args()

    source_file = args.source
    output_mbtiles = args.output
    base_name = os.path.splitext(source_file)[0]
    target_file = f"{base_name}_modified.geojson"
    
    layer_name = "transportation"
    max_zoom = "14"

    if not os.path.exists(source_file):
        print(f"Error: {source_file} not found.")
        sys.exit(1)

    print(f"Processing {source_file}...")
    with open(source_file, 'r', encoding='utf-8') as f:
        data = json.load(f)

    # Standardize transportation identification properties
    if 'features' in data:
        for feature in data['features']:
            feature['properties'] = { "class": "service", "highway": "service" }
    
    with open(target_file, 'w', encoding='utf-8') as f:
        json.dump(data, f, ensure_ascii=False)

    # Execute Tippecanoe CLI for Vector Tile generation
    cmd = ["tippecanoe", "-o", output_mbtiles, "-z14", "-l", layer_name, target_file, "--force"]
    subprocess.run(cmd, check=True)

if __name__ == "__main__":
    main()

Part B

Graph-cache Generation (GraphHopper)

1

Extract Base Infrastructure

Export map.osm from OSM Export.

2

Filter Data with Osmium

osmium tags-filter map.osm w/highway -o map-highways.osm --force
3

Merge & Topology Processing (JOSM)

• Import: Load map-highways.osm and input.geojson (Ctrl + O).

• Standardize: Select all GeoJSON features and add tag highway=service.

• Merge: Combine both layers into a single OSM layer.

• Connect: More Tools > Add nodes at intersections (Ctrl + A first).

• Save: Export as merged.osm (Ctrl + S).

4

Build & Verify Graph Environment

4.1

Prepare Binary Data

osmium cat merged.osm -o merged.osm.pbf -f pbf
4.2

Engine Build & Local Verification

Requirements:

  • Java (JRE/JDK 11+) installed.
  • graphhopper-web-5.3.jar and config-5.yml must be in the execution directory.

Run the indexing process and keep the terminal open:

java -D"dw.graphhopper.datareader.file=merged.osm.pbf" -jar graphhopper-web-5.3.jar server config-5.yml

Mandatory Verification Steps:

  1. Visit localhost:8989.
  2. Switch layer to Esri Aerial (Satellite) to check road alignments.
  3. Test routing between farm roads and main roads.
  4. Proceed only if routing is 100% accurate.
5

Final Packaging Protocol (ZIP)

After successful verification, close the Java process and package the cache:

  1. Navigate into the newly generated graph-cache folder.
  2. Select all internal files (nodes, edges, geometry, etc.).
  3. Compress directly into graph-cache.zip.
⚠ NO FOLDERS: The ZIP root must contain the data files directly.
